summary: JCode now has a single concise release-note source and a latest-package GitHub release policy.
---

## Highlights

- **One concise changelog source:** Per-version Markdown notes now drive both the app release history and GitHub release body.
- **Latest packages stay visible:** GitHub Releases keep the newest stable package and the newest prerelease package during testing.
- **Old packages are pruned:** Older GitHub Release pages and compiled assets are deleted after a newer release publishes.

## Fixes

- **Release checks fail loudly:** Missing release notes, stale generated app data, or pruning failures now block the release workflow.
